# Changelog

All notable changes to this project are documented here. The format is based on
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres to
[Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- Dedicated Maven-plugin tests cover generation, compile-root registration, missing templates,
  include/exclude patterns, and invalid directory-derived packages.
- Runtime coverage is enforced at its measured baseline, and CI publishes both runtime and core
  coverage reports.
- Release-quality build gates enforce Maven and JDK versions, reject snapshot release dependencies,
  verify all published version references, validate tags, check generated API documentation, and
  produce byte-identical binary, source, and API-documentation JARs across clean builds.
- Generated Scaladoc for the complete runtime API, linked through a stable API-reference landing
  page.
- Enforced code style: Scala sources in `nomos-runtime` and `nomos-core` are formatted with
  scalafmt (`.scalafmt.conf`), linted with scalafix (`.scalafix.conf`: `RemoveUnused` +
  `OrganizeImports`), and checked with scalastyle (`scalastyle-config.xml`, adapted from Apache
  Spark). All run in the build (`validate`/`verify`) and CI, so a violation fails the build.
  All three cover the main and test sources. Run
  `mvn scalafmt:format` to reformat.

### Fixed

- Invalid base packages, sub-packages, Scala package keywords, and empty definition sets now fail
  before generated source is written.
- Configured temporal types must be available with a public static `parse(CharSequence)` method;
  invalid configuration no longer silently uses another temporal parser.
- Maven plugin failures include per-template parse details.
- Orphan pruning does not follow directory symlinks, and custom `ScalaCodeBuilder` indentation
  widths are honored.
- Direct `GeneratedFile.writeTo` calls use the same output-directory containment checks as
  `FileWriter`.
- Published child POMs expose the repository's actual project URL.
- Contributor instructions install the local reactor before testing the isolated example.
- API documentation generation is warning-clean.
- Validation performance: the custom-validator phase now runs only for definitions whose reachable
  subtree declares validators. A validator-free template no longer pays a second whole-document
  traversal on every `validate()` call, and validator-free subtrees (and their union
  re-validation) are pruned when only part of a template uses validators.

## [0.0.1-alpha7]

### Added

- `mapType` generation setting: `$map` fields can render as `java.util.Map[String, V]` (for a
  Java-interop-friendly public surface) instead of the default immutable `Map[String, V]`. The
  concrete field type and its decode/encode wiring change; validation is unchanged.

## [0.0.1-alpha6]

### Changed

- **Breaking:** custom validators now receive a `ValidatorContext` (`node`, `root`, `path`) instead
  of the bare JSON node. `node` is the value at the validator's level, `root` is the whole document
  (so a validator can traverse up), and `path` is the JSON path. Update registrations from
  `register(name) { json => ... }` to `register(name) { ctx => ... ctx.node ... }`.

### Fixed

- Custom validators declared on a definition now run wherever that definition appears — including
  every level it is reached through a `$ref` — not only when it is the top-level definition being
  validated. Validators run only after the whole document passes structural validation.

## [0.0.1-alpha5]

### Added

- More built-in string `format` values: case-restricted GUID variants (`guidUpper`/`uuidUpper`,
  `guidLower`/`uuidLower`) alongside `guid`/`uuid`; `alpha`, `alphanumeric` and their
  `Upper`/`Lower` variants; and case-style formats `pascalCase`, `camelCase`, `snakeCase`,
  `kebabCase`, and `screamingSnakeCase` — so common identifier and GUID fields don't need
  hand-written `pattern` regexes.

## [0.0.1-alpha4]

### Changed

- **Breaking:** removed the single-valued `useOptionTypes` and `mapType` template settings. Optional
  fields are always `Option[T]` (use per-field `nullable: true` for null-based optional fields) and
  maps are always `Map[String, T]`. `listType` (`List`/`Array`) is unchanged.
- The template parser now rejects unknown top-level keys, so a typo or a removed setting fails with a
  clear error instead of being silently ignored.

## [0.0.1-alpha3]

A broad correctness, robustness, and documentation pass over the initial alpha.

### Changed

- **Breaking:** the Maven plugin now generates into `target/generated-sources/nomos` (registered as
  a compile source root) by default instead of `src/main/scala`. Point `outputDirectory` at a source
  directory only when you intend to commit generated code.
- After a successful run the plugin prunes orphaned generated files — those carrying the
  `// Code generated by Nomos. DO NOT EDIT.` header — left behind when a definition or template is
  removed. Hand-written sources are never touched.
- The `generate` goal is thread-safe and skips rewriting byte-identical files.
- Releases no longer auto-publish to Maven Central; release artifacts are verified in CI.

### Fixed

- Reject unsupported `listType`/`mapType`, conflicting same-named enums, and
  `includeDiscriminator: false` before emitting uncompilable code.
- Safe JSON number handling and exact-decimal numeric constraints.
- Resolve runtime `$ref` by package context; error on unresolved recursive refs and on
  name/output-path collisions.
- Render field defaults as type-correct literals and validate them at parse time; reject
  `useOptionTypes: false`.
- Align `datetime` validation with the decoder; make `AdapterRegistry` fail closed.
- Deterministic `variantNames`, with the full generation settings embedded in the template.
- Plugin/IO robustness: fail on write failure and on missing templates; read and write UTF-8.

### Added

- Bound validation recursion depth and precompile constraint regexes.
- Enforce the Java 8 API boundary with `--release`; add a Java 17 CI job and a coverage floor.
- Governance files and a clarified license; an authoritative docs-version gate and a docs
  drift guard.

### Documentation

- Corrected inaccuracies and completed a full docs sweep so the site and generated API reference
  match the current code.

## [0.0.1-alpha2]

Early alpha of the Nomos JSON templating engine:

- `nomos-runtime` — a dependency-free runtime (first-party JSON model, codecs, and validation)
  that generated code depends on; no third-party JSON dependency.
- `nomos-core` — the generator that turns JSON templates into Scala models with `fromJson`/
  `toJson`/`validate`, plus an embedded template for runtime validation.
- `nomos-maven-plugin` — a Maven goal that generates code from templates at build time.
- `nomos-bom` — a bill of materials to align module versions.
