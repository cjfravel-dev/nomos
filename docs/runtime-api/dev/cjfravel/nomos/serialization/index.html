  1.  object AdapterRegistry

    Registry of named (de)serialization adapters.

    Registry of named (de)serialization adapters. A field may declare an adapter to convert between its on-the-wire representation and its in-model representation, keeping output byte-compatible with existing payloads. Decode runs on parse, encode on output.

  2.  object CodecRegistry

    Registry of codecs for externally-defined types referenced by templates via $$extern:.

    Registry of codecs for externally-defined types referenced by templates via $$extern:.

    Generated code decodes/encodes an external field by looking up the codec registered under the type's fully-qualified name. Applications register a codec at startup; an unregistered name fails closed with a descriptive error rather than guessing.

  3.  object Codecs

    Decoder combinators used by generated decode methods.

    Decoder combinators used by generated decode methods. Keeping these in the runtime keeps generated code compact while remaining free of any third-party JSON library.

    A Decoder[A] turns a dev.cjfravel.nomos.json.JsonValue into either an error message or a value of type A.
