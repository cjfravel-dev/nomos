TypeDiscriminator 

case class TypeDiscriminator(fieldName: String, variants: ListMap[String, ObjectType], commonFields: ListMap[String, FieldDef] = ListMap.empty, includeInOutput: Boolean = true, variantNames: Map[String, String] = Map.empty, variantMatch: String = "exact", variantSubPackage: Option[String] = None, fallbackVariant: Option[String] = None, discriminatorEnum: Option[String] = None) extends TemplateType with Product with Serializable

Type discriminator that allows different object structures based on a discriminator field value. This creates a sealed trait with case class variants in the generated code.

fieldName

The name of the discriminator field (e.g., "type", "kind")

variants

Map of discriminator values to their respective object types (order-preserving)

commonFields

Fields that are common across all variants (order-preserving)

includeInOutput

Whether to include the discriminator field in generated case classes

variantNames

Optional mapping from variant keys to custom class names (e.g., "String" -> "StringDataContractColumn")

variantMatch

How a discriminator value is matched to a variant (default "exact")

variantSubPackage

Optional sub-package (relative to the trait's package) for the generated variant case classes; the trait stays in its own package

fallbackVariant

Optional name for a catch-all variant: an unrecognized discriminator value decodes into this case class (carrying the value and the raw object) and re-emits the preserved payload on encode, instead of failing

discriminatorEnum

Optional name for a generated enum over the discriminator values; when set, the discriminator field (on the trait and every variant) is typed as that enum instead of a raw String, and codecs map it to/from its JSON string
