Contributing to Nomos

How to set up a dev environment, build and test the modules, and land a change.

Prerequisites

  • JDK 11 — set JAVA_HOME to a Java 11 install.
  • Maven 3.9+
  • Scala 2.12.18 — managed by the build; no separate install needed.

Build & test

# build and install all modules
mvn -DskipTests install

# run the core test suite
mvn -pl nomos-core test

# rebuild the example end-to-end (regenerates from templates, then compiles + tests)
cd nomos-example && mvn test

The nomos-example module is the end-to-end proof: the plugin regenerates real models from templates, then a test suite round-trips JSON through the generated code.

Coverage

Statement coverage for nomos-core runs via scoverage:

mvn -pl nomos-core scoverage:report
# HTML report: nomos-core/target/site/scoverage/index.html

Test-driven workflow

Changes are developed test-first: add a failing spec under nomos-core/src/test/scala/dev/cjfravel/nomos/, implement until it passes, then prove anything user-visible end-to-end in nomos-example. A feature that changes generated code should have both a content test (what's emitted) and an example test (that it compiles and runs).

Code style

  • Wrap if/else branch bodies in braces.
  • Comments describe present behavior — not history, removed code, or session notes.
  • Prefer order-preserving ListMap where field/variant order matters.

Pull requests

  • Keep changes focused; update the relevant docs page when behavior changes.
  • Ensure mvn -pl nomos-core test and the example build are green.
  • Describe the user-visible effect of the change.

See Architecture for how the modules and the generation pipeline fit together.
