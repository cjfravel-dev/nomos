Template Format

A template is a JSON file describing one or more named type definitions. Its location sets the base package; its contents drive the generated case classes and validators.

File structure

A template has a top-level definitions array, plus optional project-wide settings. The base package comes from the file's path under src/main/resources/nomos/templates/.

{
  "useOptionTypes": true,
  "listType": "List",
  "definitions": [
    {
      "name": "User",
      "subPackage": "user",
      "description": "A user record",
      "template": { "id": "string" }
    }
  ]
}
Top-level fieldDefaultMeaning
definitions—Array of type definitions (required)
useOptionTypestrueOptional fields become Option[T]
listType"List"Collection type for arrays (List or Array)
mapType"Map"Type for $map (e.g. java.util.Map)
dateType / dateTimeTypejava.time.LocalDate / LocalDateTimeGenerated temporal types
fromJsonStyle"either"either or throwing (see Code Generation)

These settings are project-wide. In a multi-file project, set a value in one template and it applies everywhere — you don't repeat it in every file. If two files set the same setting to different values, generation fails with an error naming the setting, so the result is never ambiguous.

Scalar types

TemplateScala
"string"String
"number" / "double"Double
"int"Int (whole numbers only)
"long"Long (whole numbers only)
"decimal"BigDecimal
"boolean"Boolean
"date" / "datetime"java.time.LocalDate / LocalDateTime

Constraints

Use the long form { "type": ..., ...constraints } to attach validation constraints.

{
  "username": { "type": "string", "minLength": 1, "maxLength": 40, "pattern": "^[a-z0-9_]+$" },
  "email":    { "type": "string", "format": "email" },
  "age":      { "type": "int", "min": 0, "max": 150 },
  "ratio":    { "type": "number", "multipleOf": 0.5 }
}
Applies toConstraints
stringminLength, maxLength, pattern, format, enum
number / int / long / decimalmin, max, multipleOf
arrayminItems, maxItems, uniqueItems

Built-in format values: email, url, uuid, iso8601, alphaNoWhitespace, majorAndMinor. You can register more — see Validation.

Enums

A closed value set either validates a String, or generates a dedicated enum type.

// validates a String
{ "priority": { "type": "string", "enum": ["low", "medium", "high"] } }

// generates a sealed-trait enum type named Priority, and types the field as it
{ "priority": { "type": "string", "enum": ["low", "high"], "as": "enumType", "name": "Priority" } }

Arrays & maps

{
  "tags":   ["string"],
  "scores": { "type": "array", "items": "number", "minItems": 1, "uniqueItems": true },
  "labels": { "type": "array", "items": "string", "enum": ["red", "green", "blue"] },
  "settings": { "$map": "string" }
}

$map generates a Map[String, T] with unrestricted keys and validated values.

Optional, nullable & defaults

{
  "nickname": { "$optional": "string" },                       // Option[String]
  "tags":     { "$optional": ["string"], "nullable": true },   // List[String] = null (no Option)
  "retries":  { "$optional": "int", "nullable": true },        // java.lang.Integer = null (boxed)
  "verbose":  { "type": "boolean", "default": false }          // verbose: Boolean = false
}

nullable: true generates a raw nullable type instead of Option; value types are boxed (e.g. java.lang.Integer) so null compiles.

References

{
  "address": "$ref:Address",                 // another definition (any template in the project)
  "rule":    "$extern:com.example.legacy.Rule" // a hand-written type Nomos won't generate or validate
}

Self-references (recursive types) work too: a TreeNode may contain ["$ref:TreeNode"].

Type discriminators

A discriminator becomes a sealed trait with a case-class variant per value.

{
  "$type": {
    "discriminator": "kind",
    "variantMatch": "prefix",
    "variantSubPackage": "shapes",
    "commonFields": { "label": "string" },
    "variants": {
      "Decimal": { "scale": "int" },
      "circle":  "$ref:Circle",
      "square":  {}
    }
  }
}
OptionEffect
variantsMap of discriminator value → variant body (inline object or $ref)
commonFieldsFields shared by every variant
variantNamesMap several values to one class to collapse identical variants
variantMatchexact (default) or prefix — match parameterized tags like Decimal(28,8)
variantSubPackagePlace variant case classes in a sub-package
includeDiscriminatorWhether the discriminator field appears on the case classes (default true)

Reserved keys & escaping

A field literally named like a template keyword (type, enum, …) must be backtick-quoted so it's read as a field, not a directive. Nomos strips the backticks and re-escapes Scala keywords in the output.

{ "name": "string", "`type`": "string" }   // generates  `type`: String
