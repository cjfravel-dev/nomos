User Guide

Code Generation

For each definition Nomos emits a case class and a companion with parse/serialize/validate, plus a shared NomosFormats carrying the embedded template.

What gets generated

From a template at nomos/templates/com/example/models/user.json:

FileContents
com/example/models/user/User.scalaThe User case class + companion
com/example/models/NomosFormats.scalaShared mapper, embedded template, validator
…/<Enum>.scalaOne file per generated enum type

The case class & companion

case class User(id: String, username: String, email: String, age: Option[Int], roles: List[String])

object User {
  import com.example.models.NomosFormats._
  import dev.cjfravel.nomos.validation.ValidationError

  def fromJson(json: String): Either[String, User] = { ... }
  def toJson(obj: User): String = mapper.writeValueAsString(obj)
  def validate(json: String): Either[List[ValidationError], User] = { ... }
}

Discriminated unions instead generate a sealed trait, a case class per variant, and a companion whose fromJson dispatches on the discriminator value (using startsWith when variantMatch: prefix).

NomosFormats

A single NomosFormats per base package provides the Jackson ObjectMapper (Scala + Java-time modules registered), the reconstructed MultiTemplate, and a MultiValidator built from it.

object NomosFormats {
  val mapper: ObjectMapper = ...                 // Scala + JavaTime modules
  lazy val embeddedTemplate: MultiTemplate = ... // reconstructed from your template
  lazy val validator: MultiValidator = new MultiValidator(embeddedTemplate)
}

Enum files

An enumType enum generates a self-contained sealed trait with case objects and Jackson string (de)serialization wired via annotations — no extra setup.

sealed trait Priority
object Priority {
  case object Low extends Priority
  case object High extends Priority
  def fromString(s: String): Option[Priority] = ...
  def asString(v: Priority): String = ...
}

Generation options

Set these at the top level of a template (project-wide in multi-file builds):

OptionDefaultEffect
useOptionTypestrueOptional fields → Option[T] (else raw)
listType"List"List or Array
mapType"Map"e.g. java.util.Map
dateTypejava.time.LocalDatee.g. java.util.Date
dateTimeTypejava.time.LocalDateTimeTarget for datetime
fromJsonStyle"either"either or throwing

Per-field, nullable: true yields a raw nullable type (value types boxed as java.lang.Integer/Long/Double/Boolean), and default emits a case-class default argument.

fromJson style

{ "fromJsonStyle": "throwing", "definitions": [ ... ] }

either (default)

def fromJson(json: String): Either[String, T]

throwing

def fromJson(json: String): T  // throws on failure

The style applies to plain definitions and to discriminated/variantNames unions alike; validate adapts to whichever style is generated.

Helper methods

Attach derived members to a generated class with methods — emitted verbatim as instance members.

{
  "name": "User",
  "methods": ["def display: String = username + \" <\" + email + \">\""],
  "template": { "username": "string", "email": "string" }
}

File headers

Every generated file starts with a provenance header. Files tied to a template also point back to it.

// Code generated by Nomos. DO NOT EDIT.
// Source: com/example/models/user.json
package com.example.models.user
